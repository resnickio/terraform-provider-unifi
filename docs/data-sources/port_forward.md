# unifi_port_forward (Data Source)

Retrieves information about an existing UniFi port forward rule. Lookup by either id or name.

## Example Usage

### Lookup by Name

```hcl
data "unifi_port_forward" "web_server" {
  name = "Web Server"
}

output "web_server_fwd_ip" {
  value = data.unifi_port_forward.web_server.fwd_ip
}
```

### Lookup by ID

```hcl
data "unifi_port_forward" "by_id" {
  id = "60a1b2c3d4e5f67890123456"
}
```

## Schema

### Optional

- `id` (String) The unique identifier of the port forward rule. Specify either id or name.
- `name` (String) The name of the port forward rule. Specify either id or name.

### Read-Only

- `dst_port` (String) The destination port to forward from (external port).
- `enabled` (Boolean) Whether the port forward rule is enabled.
- `fwd_ip` (String) The IP address to forward traffic to.
- `fwd_port` (String) The port to forward to on the destination host.
- `log` (Boolean) Whether to log forwarded traffic.
- `pfwd_interface` (String) The WAN interface for the port forward (wan, wan2, both).
- `protocol` (String) The protocol for port forwarding (tcp, udp, tcp_udp).
- `site_id` (String) The site ID where the port forward rule exists.
- `src` (String) Source IP/CIDR restriction for the port forward.
